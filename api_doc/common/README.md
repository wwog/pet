# 通用约定

## 基础信息

| 项目 | 说明 |
|------|------|
| 基础URL | `https://api.puppy-life.com/v1` |
| 通信协议 | HTTPS（全链路） |
| 数据格式 | JSON（请求体与响应体均为 `application/json`） |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601（`2026-07-08T09:41:00+08:00`） |
| 时区 | 默认 `Asia/Shanghai`（UTC+8） |
| ID格式 | UUID v4 或 雪花ID（字符串类型，避免精度丢失） |
| 货币 | 金额单位为分（整数），展示时除以 100 |
| 重量 | 单位 kg，保留 1 位小数 |
| 距离 | 单位 m（整数），展示时按需转换为 km |

## 请求头

| Header | 必填 | 说明 |
|--------|:----:|------|
| `Authorization` | 是 | `Bearer <access_token>`（登录/注册接口除外） |
| `Content-Type` | 是 | `application/json`（文件上传除外，使用 `multipart/form-data`） |
| `X-Family-Id` | 否 | 当前家庭 ID（切换家庭后必带，后端据此做数据隔离） |
| `X-Client-Version` | 否 | 客户端版本号，如 `1.0.0` |
| `X-Device-Id` | 否 | 设备唯一标识 |

## 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": { ... },
  "timestamp": "2026-07-08T09:41:00+08:00"
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | `0` 表示成功，非 `0` 为错误码 |
| `message` | string | 描述信息，成功为 `success`，失败为错误描述 |
| `data` | object/array/null | 业务数据，成功时返回；分页接口为分页对象 |
| `timestamp` | string | 服务器响应时间 |

## 分页响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [ ... ],
    "total": 1284,
    "page": 1,
    "pageSize": 20,
    "hasMore": true
  }
}
```

## 错误码

| 错误码 | HTTP状态码 | 说明 |
|--------|:----------:|------|
| 0 | 200 | 成功 |
| 1001 | 401 | 未登录或 token 过期 |
| 1002 | 403 | 无权限（RBAC 权限不足） |
| 1003 | 404 | 资源不存在 |
| 1004 | 400 | 请求参数错误 |
| 1005 | 409 | 资源冲突（如手机号已注册） |
| 1006 | 429 | 请求过于频繁（限流） |
| 2001 | 403 | 非首席监护人，无 P12 权限 |
| 2002 | 403 | 具体权限点不足（如缺少 P4 编辑健康记录） |
| 3001 | 404 | 家庭不存在 |
| 3002 | 400 | 家庭成员已达上限（20人） |
| 3003 | 400 | 宠物数量已达上限（5只/家庭） |
| 4001 | 400 | 邀请码无效或已过期 |
| 4002 | 400 | 邀请码已被使用 |
| 5001 | 500 | 服务器内部错误 |
| 5002 | 503 | AI 服务暂时不可用 |
| 5003 | 503 | 文件存储服务不可用 |

## 权限校验机制

- 除 `auth` 类接口外，所有接口均需携带 `Authorization` token。
- 后端根据 token 解析出 `userId`，再根据 `X-Family-Id` 确定当前家庭。
- 每个接口标注所需权限点（如 `P3: 查看健康记录`），后端自动校验。
- 首席监护人默认拥有 P1-P11 全部权限 + P12 专属权限。
- 权限不足时返回 `2002` 错误码，并指明缺少的权限点。

## 家庭数据隔离

- 用户切换家庭后，所有数据查询自动限定在当前 `X-Family-Id` 范围内。
- 用户无法访问非所属家庭的数据，后端强制校验 `userId` 与 `familyId` 的归属关系。
