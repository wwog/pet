# 后台管理系统（packages/admin）实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在仓库根新增 `packages/admin` 后台管理系统（pnpm workspace + Vite + ProComponents），基于 utoipa 导出的 OpenAPI 全自动生成请求层，接入现有 auth 模块实现登录与权限，以"账号管理"示例模块走通全链路。

**Architecture:** 前端独立 Node 生态，与 Rust workspace 并列；后端 `crates/api` 增加 `--export-openapi` 导出 openapi.json；前端用 `@hey-api/openapi-ts` 生成 TanStack Query hooks；zustand 管 token/user；ProLayout + react-router v6 做布局与路由守卫。

**Tech Stack:** React 18, TypeScript 5, Vite 5, antd 5, @ant-design/pro-components, @tanstack/react-query v5, @hey-api/openapi-ts, @hey-api/client-fetch, react-router-dom v6, zustand

**真实后端接口（utoipa 注解为准）：**
- `POST /auth/register` — `{account, password, nickname}` → 201
- `POST /auth/login` — `{account, password}` → `{access_token, refresh_token, expires_in, user:{user_id, account, nickname, avatar, role}}`
- `POST /auth/token/refresh` — `{refresh_token}` → `{access_token, expires_in}`
- `GET /auth/me` — Bearer → `{user_id, account, nickname, avatar, role, created_at}`

```rust
    // 支持 `--export-openapi [path]`：仅导出 OpenAPI JSON 到文件后退出，不启动服务器
    let args: Vec<String> = std::env::args().collect();
    if args.iter().any(|arg| arg == "--export-openapi") {
        let out_path = args
            .iter()
            .position(|arg| arg == "--export-openapi")
            .and_then(|idx| args.get(idx + 1))
            .map(|s| s.as_str())
            .filter(|s| !s.starts_with("--"))
            .unwrap_or("packages/admin/openapi.json");
        let json = ApiDoc::openapi().to_pretty_json();
        std::fs::write(out_path, json)
            .expect("failed to write openapi.json");
        eprintln!("info: openapi spec exported to {out_path}");
        return;
    }
```

```typescript
import { createClient } from '@hey-api/client-fetch';
import { useAuthStore } from './stores/auth';

export const client = createClient({
  baseUrl: '/',
});

// 每个请求自动注入 Bearer token
client.interceptors.request.use((request) => {
  const token = useAuthStore.getState().accessToken;
  if (token) {
    request.headers = {
      ...request.headers,
      Authorization: `Bearer ${token}`,
    };
  }
  return request;
});

// 401 清 token 跳登录
client.interceptors.response.use((response) => {
  if (response.status === 401) {
    useAuthStore.getState().clear();
    if (window.location.pathname !== '/user/login') {
      window.location.href = '/user/login';
    }
  }
  return response;
});
```

```typescript
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export interface UserInfo {
  user_id: string;
  account: string;
  nickname: string | null;
  avatar: string | null;
  role: string;
}

interface AuthState {
  accessToken: string | null;
  refreshToken: string | null;
  user: UserInfo | null;
  setAuth: (data: { accessToken: string; refreshToken: string; user: UserInfo }) => void;
  setUser: (user: UserInfo) => void;
  clear: () => void;
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      accessToken: null,
      refreshToken: null,
      user: null,
      setAuth: (data) =>
        set({
          accessToken: data.accessToken,
          refreshToken: data.refreshToken,
          user: data.user,
        }),
      setUser: (user) => set({ user }),
      clear: () =>
        set({ accessToken: null, refreshToken: null, user: null }),
    }),
    { name: 'puppy-admin-auth' },
  ),
);
```

- [ ] **Step 1: 创建登录页**

注意：hey-api 生成的 hook 名称可能略有不同（如 `useLoginMutation` vs `useLoginService`），实施时按实际生成结果导入。下面以常见命名演示，若名称不同需相应调整 import。

```tsx
import { LoginForm, ProFormText } from '@ant-design/pro-components';
import { App as AntdApp, theme } from 'antd';
import { useNavigate } from 'react-router-dom';
import { useAuthStore } from '@/stores/auth';
import { client } from '@/client';

export default function Login() {
  const navigate = useNavigate();
  const { message } = AntdApp.useApp();
  const setAuth = useAuthStore((s) => s.setAuth);
  const { token } = theme.useToken();

  const handleSubmit = async (values: { account: string; password: string }) => {
    try {
      // 直接用 client 调用，绕过生成 hook 的命名不确定性
      const res = await client.post({
        path: '/auth/login',
        body: values,
      });
      if (res.error || !res.data) {
        message.error((res.error as { message?: string })?.message || '登录失败');
        return;
      }
      const data = res.data as {
        access_token: string;
        refresh_token: string;
        user: { user_id: string; account: string; nickname: string | null; avatar: string | null; role: string };
      };
      setAuth({
        accessToken: data.access_token,
        refreshToken: data.refresh_token,
        user: data.user,
      });
      message.success('登录成功');
      navigate('/', { replace: true });
    } catch {
      message.error('网络错误');
    }
  };

  return (
    <div style={{ minHeight: '100vh', display: 'flex', alignItems: 'center', justifyContent: 'center', background: token.colorBgLayout }}>
      <LoginForm
        title="小狗人生后台"
        subTitle="管理员登录"
        onFinish={handleSubmit}
        submitter={{ searchConfig: { submitText: '登录' } }}
      >
        <ProFormText
          name="account"
          label="账号"
          placeholder="请输入账号"
          rules={[{ required: true, message: '请输入账号' }]}
        />
        <ProFormText.Password
          name="password"
          label="密码"
          placeholder="请输入密码"
          rules={[{ required: true, message: '请输入密码' }]}
        />
      </LoginForm>
    </div>
  );
}
```
