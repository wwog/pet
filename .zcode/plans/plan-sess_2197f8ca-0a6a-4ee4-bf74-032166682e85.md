## 从路由层面区分：后台 / 客户端 / 通用

用 `utoipa_axum` 的 `.nest(prefix, router)` 按**受众**分三组挂载（同时作用于 axum 路由和 OpenAPI 文档）。

### 一、URL 分组映射

| 现路径 | 新路径 | 组 | 说明 |
|---|---|---|---|
| `POST /auth/admin/login` | `POST /admin/login` | **admin** | 仅 SuperAdmin（已有校验） |
| `POST /auth/register` | `POST /app/auth/register` | **app** | 客户端注册 |
| `POST /auth/login` | `POST /app/auth/login` | **app** | 客户端登录 |
| `GET /pets` 等全部宠物路由 | `GET /app/pets` 等 | **app** | 宠物归客户端 |
| `GET /auth/me` | `GET /common/auth/me` | **common** | 两端共用 |
| `POST /auth/token/refresh` | `POST /common/auth/token/refresh` | **common** | 两端共用 |

后台前端目前直接管理宠物 → 复用 `/app/pets/*`（符合你选的"归客户端，后台复用"）。

### 二、后端改动

**路由组装（`crates/api/src/main.rs`，导出与启动两处）**
```rust
let admin  = routes::admin_router();   // .nest("/admin", …)
let app    = routes::app_router();     // .nest("/app", …)
let common = routes::common_router();  // .nest("/common", …)
OpenApiRouter::with_openapi(ApiDoc::openapi())
    .nest("/admin", admin)
    .nest("/app", app)
    .nest("/common", common)
```

**新增 `crates/api/src/routes.rs`** — 集中组装三组 router：
- `admin_router()`：`admin_login`
- `app_router()`：`register` + `login` + `pet::router()`
- `common_router()`：`get_me` + `refresh_token`

**调整 handler 的 `#[utoipa::path]` 路径为「组内相对路径」**（`.nest` 会补前缀）：
- `admin_login`：`/auth/admin/login` → `/login`
- `register`/`login`：保持 `/auth/register`、`/auth/login`（nest 补 `/app`）
- `get_me`/`refresh_token`：保持 `/auth/me`、`/auth/token/refresh`（nest 补 `/common`）
- 宠物 handler：保持 `/pets*`（nest 补 `/app`），**无需改动**

**删除** `crates/api/src/auth/mod.rs` 里旧的单一 `router()`，改由 `routes.rs` 引用各 handler；`pet::router()` 保留不变。

**`openapi.rs`**：tags 增加分组说明（可选，纯文档）。

### 三、前端改动（`packages/admin`）

1. `vite.config.ts` 代理不变（仍 `/api` → backend，剥离 `/api`）。
2. `src/routes/login.tsx`：raw 调用 `/auth/admin/login` → `/admin/login`。
3. 重新生成 SDK：运行 `pnpm gen:api`（root）→ 重新导出 `openapi.json` 并重生成 `src/services/gen`。宠物相关调用（`listPets` 等）路径自动变为 `/app/pets*`，页面代码用的是 SDK 函数名不变，无需手改。
4. `getMe`/`refreshToken` 若未来使用，路径自动更新为 `/common/*`。

### 四、验证
- `cargo check`（后端编译）
- `pnpm gen:api` 重新生成 SDK
- `pnpm -F @pet/admin run typecheck`（前端类型）
- 人工核对导出的 `openapi.json` 里三组前缀正确

### 不改动
- 认证逻辑、`SuperAdminUser` 提取器、`AppError::Forbidden`、seed 逻辑均不变，仅调整路由挂载位置与路径前缀。
