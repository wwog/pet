## 后台品种管理 + 导出 seed JSON

### 一、需求与范围
1. **后台管理 breed**：增 + 删（不含改），仅基础字段
2. **导出 seed JSON**：API 返回 JSON 下载
3. **顺带修复**：
   - `list_pet_breeds` 硬编码 `Species::Dog` → 读 query 参数
   - 清理前端 `$petId.edit.tsx` 死字段引用（icon/pinyin/standardWeight/lifeSpan）
   - 新增 breed 时 `id` 自动生成：`{species}.{kebab-case(name)}`

### 二、后端改动

**1. Domain 层 (`crates/domain/src/pet.rs`)**
`BreedRepository` trait 新增两个方法：
```rust
async fn create(&self, breed: Breed) -> AppResult<Breed>;
async fn delete(&self, id: &str) -> AppResult<()>;
```

**2. DB 层 (`crates/db/src/pet.rs`)**
实现上述两个方法，复用现有 toasty 模式：
- `create`：先查 `find_by_id` 判重（重复 → `AppError::Conflict`），再 `toasty::create!(Breed { ... })`
- `delete`：`Breed::filter(...id.eq(id)).delete().exec(&mut db)`，按现有 pet.delete 模式

**3. API 层 — 新增 admin breed handler (`crates/api/src/pet/handler.rs` 或新文件)**

考虑放在 admin 组。由于 `pet.rs` 模块当前只挂载在 `/app` 下，新增的 admin 端点应单独挂载到 `/admin`。方案：在 `pet.rs` 内新增 `admin_router()` 返回管理员专用 breed 路由，由 `routes.rs` 的 `admin_router()` merge。

新增 3 个 handler：

| Handler | Method & Path | 说明 |
|---|---|---|
| `create_breed` | `POST /admin/breeds` | 创建 breed，id 自动生成 |
| `delete_breed` | `DELETE /admin/breeds/{breedId}` | 删除 breed |
| `export_breeds` | `GET /admin/breeds/export` | 返回完整 JSON（按 species 分组的 `{species: [...]}` 或合并数组） |

- 三个 handler 均用 `SuperAdminUser` 提取器保护
- `create_breed` 请求体 DTO：`CreateBreedRequest { species, name, name_cn, size_category?, coat_type?, origin? }`；后端校验 species 合法、name 非空，生成 id 后调用 `repo.create`
- `export_breeds`：从 DB 拉全量 breeds（需在 trait 加 `find_all` 或复用 `search` 大分页），按 species 分组序列化为 seed JSON 格式，返回 `application/json`，文件名 `Content-Disposition: attachment; filename="breeds.json"`

**4. `list_pet_breeds` 修复 (`crates/api/src/pet/handler.rs:655`)**
将硬编码 `Species::Dog` 改为从 query 读取 `species` 参数（`Species::from_str` 校验，缺失默认 Dog 或返回 400）。`BreedQueryParams` 增加 `species: Option<String>` 字段。

**5. `routes.rs`**
`admin_router()` 合并 `pet::admin_router()`：
```rust
pub fn admin_router() -> OpenApiRouter<SharedState> {
    OpenApiRouter::new()
        .routes(routes!(auth::handler::admin_login))
        .merge(pet::admin_router())
}
```

`pet.rs` 新增：
```rust
pub fn admin_router() -> OpenApiRouter<SharedState> {
    OpenApiRouter::new()
        .routes(routes!(handler::create_breed))
        .routes(routes!(handler::delete_breed))
        .routes(routes!(handler::export_breeds))
}
```

**6. 需要在 `BreedRepository` 加 `find_all`**（用于 export）：
```rust
async fn find_all(&self) -> AppResult<Vec<Breed>>;
```
DB 实现：`Breed::all().exec(&mut db)`。

### 三、前端改动（`packages/admin`）

**1. 重新生成 SDK**：`pnpm gen:api` 自动生成 `createBreed` / `deleteBreed` / `exportBreeds` + 修复后的 `listPetBreeds`（带 species 参数）。

**2. 新增品种管理页面 `src/routes/_authed/breed/index.tsx`**
- 表格列出 breeds（用 `listPetBreeds` 带 species 筛选切换 dog/cat）
- 顶部「新增品种」按钮 → 弹出/跳转表单填写基础字段
- 每行「删除」按钮（二次确认）
- 「导出 seed JSON」按钮 → 调用 `exportBreeds` 下载

**3. `Sidebar.tsx`**：将品种管理菜单项从「待接入」改为可用，指向 `/breed`。

**4. 清理 `$petId.edit.tsx` 死字段**：
- 删除 `breed.icon`、`breed.pinyin`、`selectedBreed.standardWeightMin/Max`、`lifeSpanMin/Max` 相关渲染代码（约 411-430 行）
