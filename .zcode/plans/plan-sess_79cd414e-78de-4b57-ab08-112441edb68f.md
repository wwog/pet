# Admin 移动端适配方案

## 目标
为 admin 管理后台补齐移动端体验,断点 md(768px)。桌面端布局保持不变,移动端改为抽屉式导航。

## 改动清单

### 1. 新建 `src/lib/use-is-mobile.ts`
基于 `matchMedia('(max-width: 767px)')` 的 hook,返回 `boolean`。需要 JS 状态来控制抽屉 open/close、路由切换后自动关闭、遮罩点击关闭等交互,纯 CSS `md:hidden` 无法实现。

### 2. 改写 `src/components/layout/Sidebar.tsx`
- 桌面端(md+):保持 `w-60 shrink-0` 常驻,行为不变
- 移动端(<md):fixed 定位抽屉,`translate-x` 滑入滑出 + 半透明遮罩
- 新增 props: `{ mobileOpen: boolean; onMobileClose: () => void }`
- 抽屉打开时锁 body 滚动
- 点击导航项后调用 `onMobileClose` 自动收起
- 复用现有 `MODULES` 数据,不重复维护

### 3. 改写 `src/components/layout/Topbar.tsx`
- 移动端:左侧加汉堡按钮(触发 `onMenuClick`)
- 小屏精简 `ADMIN / 管理后台` 文案显示
- 桌面端布局不变
- 新增 prop: `{ onMenuClick: () => void }`

### 4. 改写 `src/components/layout/AppShell.tsx`
- 用 `useState` 管理 `mobileNavOpen`
- 桌面:`flex` [Sidebar 常驻 | 右侧列(Topbar + main)]
- 移动:右侧列全宽 [Topbar(含汉堡) | main],Sidebar 作为 fixed 抽屉叠加
- main padding 从 `p-6` 调整为 `p-4 md:p-6`
- 监听 ESC 键关闭抽屉

### 5. 微调 `src/routes/login.tsx`
- 外层补 `px-4`,避免小屏卡片被边缘挤压
- 不改结构

### 不改动
- `dashboard.tsx`(已有 `md:grid-cols-3`)
- 路由结构、`services/gen` 生成层、`MODULES` 列表内容
- 不引入 UI 组件库,保持手写 + Tailwind 风格

## 验证
- `pnpm -F @pet/admin run build` 确保 TS 编译通过
- `pnpm -F @pet/admin run dev` + 浏览器设备模拟,检查:
  - 移动端汉堡按钮可见、抽屉滑入滑出
  - 遮罩点击关闭、ESC 关闭
  - 点击导航项后抽屉自动关闭、路由正常跳转
  - 桌面端布局与改动前一致
