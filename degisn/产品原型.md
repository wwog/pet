# “小狗人生”产品需求文档（PRD）


## 第一章 产品概述

### 1.1 产品名称
小狗人生（Puppy Life OS）

### 1.2 产品定位
一款面向现代养宠家庭的**AI原生宠物生活管理平台**。产品核心定位不仅是记录工具，更是家庭的“AI宠物管家”与“毛孩子数字生命档案”。

### 1.3 核心理念
> **“用科技的温度，延伸爱的刻度。”**

通过AI能力与家庭协作机制，将零散的养宠信息整合为可视化、可追溯、可协作的宠物全生命周期档案。让爱宠的每个高光时刻都被妥善安放，每一次健康风险都被提前预判，让养宠成为全家共同参与的温暖仪式。

### 1.4 目标用户
- **核心用户**：25-40岁城市养宠人群（犬主人）。
- **家庭成员**：伴侣、父母、子女、合租室友等共同照料宠物的多角色人群。
- **衍生用户**：临时寄养人、上门喂养服务人员（仅限使用访客模式）。


## 第二章 用户与家庭体系

### 2.1 用户系统（轻量级）
| 条目 | 设计说明 |
|------|----------|
| **注册登录** | 仅支持“手机号+验证码”或“微信一键登录”，不设独立密码。 |
| **用户ID** | 每个手机号绑定唯一用户ID。 |
| **多家庭归属** | 一个用户可以同时属于**多个家庭**（如自己家、父母家），并在不同家庭中担任**不同角色标签**。 |
| **默认家庭** | 注册成功后，系统自动创建以“XX（用户昵称）的家”命名的默认家庭。 |

### 2.2 家庭（Group）架构
**家庭是宠物的所有权主体。** 宠物不挂载在个人名下，而是挂载在家庭名下。

| 属性 | 说明 |
|------|------|
| **绑定宠物** | 每个家庭可绑定 1 ~ 5 只宠物。 |
| **家庭信息** | 家庭名称、家庭头像、所在城市（用于本地化服务推荐）。 |
| **家庭容量** | 暂定每个家庭最多容纳 20 名成员。 |

### 2.3 家庭创建与加入流程

| 场景 | 流程详情 |
|------|----------|
| **首次注册** | 填写昵称后自动生成默认家庭，用户自动成为该家庭的**首席监护人**。可立即添加宠物或稍后操作。 |
| **邀请家人** | 首席监护人通过“家庭管理”生成**6位数字邀请码**或**微信分享链接**。 |
| **加入与审批** | 被邀请人注册/登录后输入邀请码，**自选角色标签**并**预览待赋予的权限**，提交申请后需**首席监护人审核通过**正式生效。 |
| **多家庭切换** | 首页左上角显示当前家庭名称及成员头像缩略图，下拉可快速切换。切换后全站数据同步切换。 |

### 2.4 角色标签系统（仅用于身份展示，与权限解耦）
角色不再绑定权限，仅作为**操作留痕时的显示名称**和**情感化身份标识**。

**预设角色标签库**（可自定义编辑）：
- 首席监护人（特殊标识，不可更改，仅限1人）
- 奶爸、奶妈
- 爷爷、奶奶
- 管家、助理
- 健康顾问、家庭摄影师
- 亲友、临时照料人
- 支持**自定义角色名称**（如“干爹”、“御用铲屎官”）

> **注意**：“首席监护人”既是角色标签，也是系统硬编码的家庭最高管理者身份，包含特殊的不可转让权限（详见第三章）。


## 第三章 权限系统（柔性RBAC）

### 3.1 设计核心理念
- **权限与角色完全解耦**。角色仅用于展示，权限逐项独立开关。
- **首席监护人**拥有除“删除家庭/移除自己”之外的全部操作权力，并可自由为每位成员分配任意权限组合。
- 所有权限调整**实时生效**，并记录操作日志。

### 3.2 权限清单（12个独立权限点）
| 编号 | 权限名称 | 说明 |
|------|----------|------|
| P1 | 查看宠物基础档案 | 姓名、品种、生日、性别、绝育状态等 |
| P2 | 编辑宠物基础档案 | 修改上述所有基础信息 |
| P3 | 查看健康记录 | 疫苗、驱虫、体重、过敏史、病历 |
| P4 | 编辑/新增健康记录 | 添加或修改任何健康数据 |
| P5 | 查看每日事件/日记 | 浏览时间线动态与日记内容 |
| P6 | 编辑/新增每日事件 | 记录、编辑或删除事件/日记 |
| P7 | 查看日程列表 | 浏览所有待办与周期性日程 |
| P8 | 管理日程 | 添加、修改、删除、标记完成日程 |
| P9 | 查看相册 | 浏览所有照片/视频 |
| P10 | 管理相册 | 上传、删除、编辑相册内容 |
| P11 | 使用AI对话 | 发起AI咨询并查看历史对话 |
| P12 | 管理家庭成员 | **硬编码权限**，仅首席监护人天生拥有，**不可授予其他成员**（含邀请、移除、调整他人权限） |

### 3.3 权限分配交互界面
在“家庭管理 → 成员列表 → 点击成员”进入权限编辑页：

- 显示当前成员的角色标签（可独立编辑）。
- 12项权限均以**开关（Toggle）** 形式呈现，逐项自由勾选。
- 底部提供 **“快速应用模板”** 下拉菜单（如“仅查看”、“奶爸全选”、“管家模板”），一键套用后可继续手动微调。
- 权限变更后，被修改人将收到站内通知；若其正在执行被移除权限的操作，**系统强制中断并提示刷新**。

### 3.4 首席监护人特殊规则
| 规则 | 说明 |
|------|------|
| **默认权限** | 拥有 P1 ~ P11 全部权限（P12为其专属特权）。 |
| **不可执行操作** | ① 不可解散家庭（需先转让首席监护人身份）；② 不可移除自己（需先转让身份）。 |
| **身份转让** | 可将首席监护人身份转让给任意家庭成员，转让后自己降级为普通成员，失去P12权限，但其余权限保留不变。 |
| **权限覆盖** | 对任何成员的权限调整无需对方确认，直接生效。 |

### 3.5 权限矩阵速查表

| 权限点 | 首席监护人 | 普通成员（视具体分配） |
|--------|:----------:|:----------------------:|
| P1 ~ P11 | ✅ 全选 | ⚠️ 逐项自定义（可为全选/全不选/任意组合） |
| P12（管理成员） | ✅ 专属 | ❌ 不可被赋予 |


## 第四章 新用户引导与宠物档案创建

### 4.1 无宠状态（空状态主页）
- 展示**2.5D动态Lottie动画**：小狗在云端草坪嬉戏，随手机倾斜产生视差。
- 支持轻触交互：点击小狗跳跃、转圈，激发添加兴趣。
- 仅设一个核心按钮 **“+ 迎接新家人”** ，点击进入沉浸式引导。

### 4.2 添加宠物三步引导（“能用选的绝不输”）

**第一步：基础身份（仅1个必填文本）**
- **爱称**（唯一文本输入框，自带emoji推荐）。
- 性别（卡通图标点选：男/女）。
- 出生日期（提供“大概年份/月份”滑动条，适配领养未知情况）。

**第二步：品种与外貌（图形化点选）**
- 品种库集成 **AKC + FCI** 全犬种数据库，支持中文拼音首字母检索（如输入“J”显示金毛、吉娃娃），并展示该品种**标准体重范围与寿命参考**。
- 毛色：12种基础色板 + 花纹贴纸（陨石、虎斑等）。
- 绝育状态：三态切换开关（已绝育 / 未绝育 / 计划中）。

**第三步：性格与偏好（游戏化标签云）**
- 提供20+个标签（社牛、护食、爱游泳、怕生等），点击即添加，构建初始性格画像，为AI对话提供个性化基础。


## 第五章 核心功能模块详解

### 5.1 今日·AI简报（首页默认Tab）
- **功能**：基于时间、天气、品种年龄，AI生成每日摘要。
- **展示内容**：今日遛弯时长建议、昨日睡眠分析（若接入智能设备）、今日待办优先级排序、一句AI暖心寄语。

### 5.2 云相册·时光机
- **基础**：无限量原画质备份，AI自动剔除模糊/重复照片。
- **亮点① AI标签检索**：自动识别动作（飞盘/睡觉/呕吐）与表情（开心/委屈），支持语义搜索（如“去年冬天在公园接飞盘”）。
- **亮点② 时光回忆录**：每月自动生成15秒高光短视频，支持自定义配乐。

### 5.3 日历代办·智能日程
- **基础**：记录驱虫、疫苗、洗澡、刷牙、粮草购买等周期性事件。
- **亮点① 智能提醒引擎**：根据上次记录自动计算下次提醒日，并推送附近合作医院或线上药房链接。
- **亮点② 家庭共享协作**：多人共同编辑日程，谁遛弯、谁喂饭责任清晰。

### 5.4 AI对话·宠物翻译官
- **行为解读**：上传视频或描述行为，AI分析背后原因（如分离焦虑、早期疼痛表现）。
- **紧急分诊**：描述症状后，AI给出紧急等级（绿/黄/红）及就医前预处理建议，一键导航至最近24小时急诊医院。
- **拟人化聊天**：切换至“模拟宠物口吻”模式，用狗狗第一人称回应用户，增加趣味性。

### 5.5 健康档案·疫苗日记升级版（电子病历本）
- **免疫记录**：疫苗名称、批号、接种医院、下次加强针倒计时。
- **驱虫记录**：体内/体外区分，支持药品外包装拍照存档。
- **体重/体况曲线**：定期录入，AI自动评估肥胖风险并给出饮食建议。
- **过敏史与用药禁忌**：红色高亮警示全体家庭成员。
- **基因检测报告**：支持上传PDF并AI提取关键信息解读。

## 第七章 操作留痕与家庭动态墙

### 7.1 全局动态墙（宠物主页顶部新增Tab）
以 **“谁 + 做了什么 + 何时”** 格式记录所有关键操作，示例：

| 操作类型 | 动态展示文案 |
|----------|-------------|
| 更新档案 | “奶爸 张三 更新了豆豆的体重：12.5kg → 13.0kg” |
| 记录事件 | “奶妈 李四 记录了豆豆今天在公园学会了‘握手’” |
| 完成日程 | “管家 王五 标记了‘体内驱虫’任务为已完成” |
| 上传照片 | “爷爷 赵六 上传了3张新照片” |
| AI咨询 | “奶爸 张三 咨询了呕吐问题，AI建议观察至明日” |
| 权限变更 | “首席监护人 刘七 调整了 李四 的权限：新增‘管理日程’” |
| 角色变更 | “首席监护人 张三 将 李四 的角色标签从‘奶妈’改为‘管家’” |

### 7.2 交互与筛选
- 动态默认折叠，点击展开详情（药品品牌、照片预览等）。
- 支持按**成员**或**操作类型**筛选。
- 重要医疗操作（疫苗/用药）自带红色角标，确保不被信息流淹没。

### 7.3 责任可视化
- 宠物主页右上角展示 **“本月活跃成员”** 小卡片。
- 统计每位成员本月记录次数，并赋予趣味称号（如“最佳铲屎官”、“遛弯冠军”）。


## 第八章 数据安全与隐私

| 维度 | 措施 |
|------|------|
| **传输与存储** | 全链路HTTPS，云相册与健康数据采用端到端加密（E2EE）。 |
| **数据导出** | 支持用户随时导出全部数据（PDF/CSV格式）。 |
| **AI隐私** | 明示AI对话记录仅用于当次咨询，默认不用于模型训练；若用于优化需单独弹窗授权。 |
| **权限隔离** | 不同家庭间数据完全隔离，用户切换家庭后无法访问前一家庭数据。 |
| **账号注销** | 提供“注销账号”功能，注销后清除所有个人身份信息，但家庭内操作记录脱敏保留（保留“已注销用户”标识）。 |


## 第九章 非功能性需求

| 项目 | 要求 |
|------|------|
| **性能** | 首页加载时间 < 1.5秒；相册缩略图采用CDN加速；AI对话响应首字延时 < 2秒。 |
| **兼容性** | 支持 iOS 13+ 及 Android 8.0+；适配各种屏幕尺寸（含折叠屏）。 |
| **推送** | 集成极光/个推，确保重要日程提醒、权限变更通知的到达率 > 99%。 |
| **国际化** | 首版支持简体中文，预留多语言国际化框架（后续拓展）。 |
